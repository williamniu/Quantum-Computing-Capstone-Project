# Capstone — Predicting Macroeconomic Indicators with Classical and Quantum-Enhanced Models

UChicago MS-ADS capstone · Spring 2026 · Sponsor: **Bank of Montreal (BMO)**

## What this project is

BMO joined the IBM Quantum Network in 2025 and launched an Institute for Applied AI
& Quantum. They want to know, concretely, **when (and whether) quantum-enhanced
machine-learning methods can beat classical state-of-the-art at forecasting U.S.
macroeconomic indicators**.

Two quantum techniques are in scope:

- **Quantum Reservoir Computing (QRC)** — features pass through a random quantum
  circuit that projects them into a high-dimensional Hilbert space, then a simple
  classical readout is trained on the resulting embeddings.
- **Quantum Enhanced Signature Kernels (QESK)** — a signature-kernel classifier where the
  feature map is implemented by a parameterized quantum circuit.

We replicate and extend **Konduri & Li (2024)**, a FRED-MD benchmarking study that
compares classical ML methods on U.S. macro forecasting, and add the two quantum
models on top. Target indicators (per the sponsor's problem statement):

- **Regression:** `INDPRO`, `PAYEMS`, `CPIAUCSL`, `S&P 500` at horizons *h* ∈ {1, 3, 6, 12}
  
Deliverables: the best classical baseline we can build + a QRC model + a QESK model,
all evaluated on identical data and identical expanding-window folds so scores are
directly comparable. Final showcase: Saturday, 2026-12-05.

## How the data pipeline works

`notebooks/01_data_prep.ipynb` is the shared substrate for every downstream model.
It:

1. Loads **FRED-MD** (McCracken & Ng, March 2026 vintage) — 126 variables × 806 months
2. Fetches **NBER USREC** recession indicator from FRED (with offline fallback)
3. Cleans (drops high-missing and late-start series, forward-fills publication-lag tails)
4. Applies the **McCracken-Ng transformation codes** to produce a stationary panel
5. Verifies stationarity with the **Augmented Dickey-Fuller** test
6. Builds four parallel feature pathways:
   - **Track A** — the full 123-dim stationary panel
   - **Track B** — 6 sponsor-curated leading indicators (incl. T10Y3M level spread)
   - **Track C** — an *unfitted* LASSO-selection pipeline (fit per training fold)
   - **Track D / D-mini** — *unfitted* PCA pipelines (~30 factors / 15 factors)
7. Builds multi-horizon targets — h-step log growth for the 4 regression indicators,
   forward-shifted USREC for classification
8. Builds a **QSK path tensor** (sliding 24-month windows with time augmentation)
9. Defines **39 expanding-window folds** per Konduri & Li (2024)
10. Exports everything to `data/processed/`

### The design decision that matters most

Tracks C and D ship as **unfitted `sklearn.Pipeline` objects**, not as pre-reduced
feature panels. Feature selection (LASSO) and dimensionality reduction (PCA) both
*fit* on data; fitting them once on the whole history would silently encode
information from what will later become the test set. Downstream modeling
notebooks call `.fit()` on each training fold separately, keeping test data
completely unseen. A leakage sanity check in §7.3 confirms this works —
different folds select different features.

## Running the notebook

```bash
# optional: override the project root on a different machine
export BMO_CAPSTONE_ROOT="/path/to/capstone-quantum"

# from the repo root, run top-to-bottom
jupyter nbconvert --to notebook --execute --inplace notebooks/01_data_prep.ipynb

# re-export the HTML if you edit the notebook
jupyter nbconvert --to html notebooks/01_data_prep.ipynb
```

Dependencies: `pandas`, `numpy`, `scikit-learn`, `statsmodels`, `matplotlib`,
`seaborn`, `pandas_datareader`, `pyarrow`, `nbformat`, `nbconvert`.
Tested with `pandas 2.3`, `numpy 2.4`, `sklearn 1.8` (exact versions pinned in
`data/processed/metadata.json`).

Runtime: ~2 minutes on a laptop.

## Next steps — modeling notebooks

With the substrate in place, the project continues in three parallel notebooks
(one per model family). Each loads from `data/processed/` and uses the same folds:

- `02_classical_baseline.ipynb` — Elastic net / random forest / XGBoost / factor
  models on Tracks A/B/C/D. Produces the benchmark numbers every quantum variant
  must beat.
- `03_qrc.ipynb` — Quantum Reservoir Computing wired to Track D-mini (Qiskit Aer
  for local simulation, IBM Quantum processors via BMO's instance for final runs).
- `04_qsk.ipynb` — Quantum Signature Kernel on the Track B path tensor.
